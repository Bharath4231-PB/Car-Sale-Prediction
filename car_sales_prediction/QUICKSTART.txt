ISSUE: Python: command not found
SOLUTION: 
   1. Ensure Python is installed (python.org)
   2. Add Python to PATH environment variable
   3. Restart terminal/command prompt

ISSUE: pip: command not found
SOLUTION:
   1. Use: python -m pip install -r requirements.txt
   2. Or ensure Python was added to PATH

ISSUE: ModuleNotFoundError (Flask, pandas, etc.)
SOLUTION:
   1. pip install -r requirements.txt
   2. Verify file path is correct
   3. Use: python -m pip instead of pip

ISSUE: MySQL Connection Error
SOLUTION:
   1. Start MySQL: net start MySQL80
   2. Check username: root
   3. Check password: ben10
   4. Check database exists: mysql -u root -pben10 -e "SHOW DATABASES;"

ISSUE: database.sql: No such file
SOLUTION:
   1. Navigate to correct directory:
      cd e:\car sales prediction\car_sales_prediction
   2. Use absolute path:
      mysql -u root -pben10 < e:\car sales prediction\car_sales_prediction\database.sql

ISSUE: model.pkl: No such file (when running app)
SOLUTION:
   1. Train model first: python train_model.py
   2. Ensure you're in correct directory
   3. Check model.pkl file was created

ISSUE: 404 errors for CSS/JavaScript
SOLUTION:
   1. Verify static/ folder exists
   2. Check file paths in templates
   3. Restart Flask server
   4. Clear browser cache (Ctrl+Shift+Del)

ISSUE: Login fails (Invalid credentials)
SOLUTION:
   1. Ensure database is set up: mysql -u root -pben10
   2. Check users table: SELECT * FROM users;
   3. Try admin account: admin / admin123
   4. Register new account

ISSUE: Prediction shows "Error"
SOLUTION:
   1. Check model is trained: python train_model.py
   2. Verify all input fields are filled
   3. Check console for error messages
   4. Restart Flask server

ISSUE: 500 Internal Server Error
SOLUTION:
   1. Check Flask console for error message
   2. Common causes:
      - Database not connected
      - Model file missing
      - Syntax error in Python
   3. Restart server: python app.py
